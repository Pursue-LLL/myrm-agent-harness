# mcp/

## Overview
MCP toolkit: client/transport management, agent tool discovery, schema tolerance, security/OAuth, and a persistent, self-healing connection pool (one warm `ClientSession` per server, reused across setup and runtime, reconnecting transparently on transport breaks with transport-aware keepalive).

## File & Submodule Index

| File | Role | Description | I/O/P |
|------|------|-------------|-------|
| __init__.py | Package | MCP toolkit entry point. Aggregates client management, agent tool fetching, connection pooling, | ✅ |
| agent.py | Core | MCP tool discovery layer (`MCPAgent`). **Not** `myrm_agent_harness.agent` — filename is MCP-domain naming only. Orchestrates multi-server tool discovery with parallel fetching. `process_session_tools()` is the single post-processing chain (filter→**prefix**→description limit→sanitize→timeout→annotations) shared by persistent-session actors and one-shot enumeration. **Server-prefix isolation** (`_prefix_tool_names`): every MCP tool is renamed to `mcp__{sanitized_server}__{tool}` — prevents silent collisions when multiple servers expose identical names, eliminates permission bypass when an MCP name matches a built-in, and makes tool origin transparent in logs/audit. Filtering (`_apply_tool_filter`) runs *before* prefixing so `tool_include/exclude` configs use original names. **Content block coercion** (`_coerce_content_block`): guarantees only LLM-safe types (`text`, `image`) reach the API — `file` blocks (from `ResourceLink`), malformed `image`, and unknown types are gracefully degraded to `text`, preventing Anthropic/OpenAI 400 errors and session history poisoning. **Upstream fault tolerance**: `_timeout_wrapper` catches adapter-layer exceptions (`NotImplementedError` for `AudioContent`, `ValueError` for unknown types) from `langchain_mcp_adapters`, returning readable error messages instead of crashing. `_normalize_mcp_result` applies coercion to every block, preserves valid image blocks as native `list[dict]` for ToolMessage multimodal pipeline, and extracts `structuredContent` from MCP artifacts as supplementary text blocks; text-only results fall back to plain `str`. `_extract_mcp_app_metadata` detects ext-apps `_meta.ui.resourceUri` in artifacts; `_emit_mcp_app_event` emits an SSE event via progress_sink to notify the frontend of embeddable MCP App UIs. | ✅ |
| client.py | Core | MCP client management layer. Handles MCP server connection setup, transport config conversion, and TLS/mTLS injection for HTTP transports via an `ssl.SSLContext` (load_cert_chain) + httpx_client_factory — supports encrypted client keys, CA bundle as PEM file or capath directory, and emits actionable errors (no silent key drop, no DeprecationWarning). | ✅ |
| config.py | Config | MCP Configuration. Supports ssl_verify / client_cert / client_key / client_key_password for mTLS, first-class `headers` field for SSE/streamable_http with `{{secret:KEY}}` template reference, per-server `tool_include`/`tool_exclude` whitelist/blacklist fields + `should_register_mcp_tool()` filter function, `sanitize_mcp_name_component()` for safe tool-name generation (replaces non-`[A-Za-z0-9_]` chars with `_`, collapses consecutive underscores, strips leading/trailing underscores, and falls back to `'unnamed'` for empty results — preserving `__` delimiter uniqueness), and `parse_mcp_tool_name()` / `is_mcp_tool_name()` for unambiguous `mcp__{server}__{tool}` prefix parsing. | ✅ |
| connection_manager.py | Core | MCP connection pool. Keys one warm connection per config-hash; each connection owns one `MCPSessionActor` per server (single persistent session, spawned once). Loop-aware (rebuilds on event-loop change), health-checked, TTL-recycled by actor last-activity. Actors self-heal transport breaks internally, so a pool rebuild is only the last resort once an actor's reconnect budget is exhausted. Exposes `tools_by_server` / `instructions_by_server` for setup, `call()` for direct tool invocation, and `read_resource()` for MCP Apps UI resource fetching. | ✅ |
| session_actor.py | Core | Persistent, self-healing MCP session actor. Enforces `scan_mcp_runtime_surface` before first tool freeze and on dynamic tool refresh. Owns one long-lived `ClientSession` in a dedicated owner task: initializes once, serves all calls serially via an internal queue (eliminating per-call subprocess spawn / re-initialization). Supports both tool invocations (`call`) and resource reads (`read_resource` for MCP Apps ext-apps UI content). On a transport break it reconnects in place with bounded backoff — rebinding executable tools to the fresh session while the agent-facing proxy objects stay stable (prefix-cache safe). **Dynamic tool discovery**: listens for `notifications/tools/list_changed` via `message_handler` injected into `ClientSession`; the notification enqueues a `_REFRESH_SIGNAL` so `_refresh_tools` runs serialised in the owner task (zero locks, zero concurrency issues) — updates the execution-layer tool map while leaving prompt-facing proxies frozen (cache-stable). Both initialization and dynamic refresh paths are protected by `connect_timeout` to prevent owner-task deadlock if a server becomes unresponsive. `_resolve_tool` supports backward-compatible fallback: if a caller uses the original unprefixed name, the actor tries adding the `mcp__{server}__` prefix automatically. Remote transports (SSE/streamable_http) get a periodic in-band keepalive ping so idle connections behind LB/NAT are never silently dropped; stdio (local pipe) is left unprobed. | ✅ |
| errors.py | Core | MCP error handling utilities; `MCPRuntimePostureError` for runtime scan blocks. | ✅ |
| lifecycle.py | Core | MCP lifecycle management for connection pool startup/shutdown. | ✅ |
| security.py | Core | MCP security primitives: SSRF URL validation, response size limits, OSV malware advisory, and re-exports of static scan DTOs/APIs from `config_scan.py`. | ✅ |
| config_scan_patterns.py | Core | Compiled regex patterns for MCP static/runtime security scanners (risk profiles, exfil URLs, identifier + text injection with `[\s_]+`, sensitive paths including `.gnupg/.kube`). | ✅ |
| config_scan.py | Core | Static MCP configuration pre-flight scanner and runtime surface (instructions, tool names, tool descriptions) prompt-injection scanner. Identifier rules use `[\s_]+` variants aligned with `sanitize_mcp_name_component`. Reuses `leak_detector` for credential exposure; <10ms typical. | ✅ |
| oauth.py | Core | MCP OAuth 2.0 + PKCE authentication provider. Implements `MCPAuthProvider` Protocol for remote MCP servers requiring OAuth authorization. Storage-agnostic — delegates token persistence to the business layer via `MCPOAuthTokenStore` Protocol. | ✅ |
| schema_utils.py | Core | MCP Schema Utilities. Provides $ref resolution, cache-stable canonicalization (deterministic key ordering for prompt prefix cache stability), deep-nesting flattening (dot-path notation for LLM compatibility), and dynamic type coercion. Full tolerance chain: $ref→canonicalize→flatten→coerce→nest. | ✅ |
| transport.py | Core | Executor-backed MCP Stdio Transport (`ExecutorStdioTransport`): nested-sandbox stdio capability (bwrap/docker) with stdout JSON filtering, stderr draining + log rotation, robust termination. Standalone capability, not yet wired into the live pool path. | ✅ |
